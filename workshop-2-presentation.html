🧠 Mental Model: The Setup Tax

OpenClaw was built for tinkerers and hackers.

It was not meant to be easy to set up. We're making it as smooth as we can.

  • 🪪 It's like onboarding a new employee — they need access to everything
  • 😤 Many things don't work in one click — expect frustration
  • ✅ But once it's done, it's done — you never set it up again
  • 🚪 This is where most people give up
  • 🤝 That's why we're here
The setup is the hardest part. Today might be frustrating. That's normal. Push through it — the payoff is an employee that works 24/7.

🔄 Mental Model: Close the Loop

Don't accept “I tried but it didn't work”

Train your agent to test itself until the task is actually complete.

❌ Bad

  • “Set up email”
  • Agent configures IMAP
  • “Done!”
  • ...but it doesn't actually work

✅ Good

  • “Set up email and send me a test email to confirm it works”
  • Agent configures IMAP
  • Agent sends test email
  • Agent verifies delivery
  • “Done — check your inbox”

Always ask: “...and verify it works”

⏰ Cron Jobs & Scheduling

Make agents work while you sleep

⏰ Why Scheduling Matters

Without scheduling: you ask, it answers.

With scheduling: it works on its own.

Heartbeat (Workshop 1)

  • Runs every N minutes
  • Batch multiple checks
  • Good for periodic monitoring

Cron Jobs (NEW)

  • Exact timing — "9am Monday"
  • One-shot reminders
  • Isolated from main session
  • Can use different models

⏰ Cron Job Types

⏱️ One-shot

"Remind me in 20 minutes"

Fires once, then done

🔁 Recurring

"Every day at 6am, check my emails"

Repeats on a schedule

📅 Cron expression

"Every weekday at 9am"

0 9 * * 1-5
Every morning at 8am, check my Gmail for any urgent emails and send me a summary on Telegram.

⏰ Real Examples

  • 📧 8am daily — Email digest of overnight messages
  • 📊 Monday 9am — Weekly revenue report
  • 🐦 Every 4 hours — Check Twitter mentions
  • 📰 6pm daily — Hacker News top stories summary
  • 💡 6am daily — LinkedIn post ideas based on trending topics

The agent becomes a proactive employee, not a reactive tool.

🔐 1Password Integration

Secure credential management

🔐 Why 1Password?

Your agent needs credentials. Don't put them in text files.

❌ Bad

  • Passwords in TOOLS.md
  • API keys in chat history
  • Creds in environment variables

✅ Good

  • Agent calls op CLI
  • Fetches creds on demand
  • Supports TOTP (2FA codes)
  • Requires your approval via app

🔐 How It Works

1
Install op CLI

brew install 1password-cli

2
Enable desktop integration

In 1Password app settings

3
Agent runs op signin

You approve on your device

4
Fetches secrets securely

Passwords, TOTP, API keys

Set up 1Password CLI integration so you can securely fetch my credentials when needed. I have the 1Password desktop app installed.

⌨️ Chat Commands

Type these directly in Telegram to control your agent

CommandWhat it does
/modelsSwitch AI model (Sonnet → Opus, GPT-4, etc). Cheaper or smarter on demand.
/restartRestart the agent. Clears stuck state, reloads workspace files.
/contextShow current context window usage. See how much memory the agent is using.
/statusSession status — model, usage, cost, uptime.
/reasoningToggle extended thinking. Agent "thinks harder" on complex tasks.
/tasksView running background tasks — cron jobs, sub-agents, long-running work.

💡 Pro tip: use /models to switch to a cheaper model for simple tasks, and Opus for complex ones.
